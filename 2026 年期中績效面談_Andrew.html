02. 上半年核心成果：AI 流程創新與品質優化

拒絕盲目敲代碼，用 AI 提高產出上限與系統良率

01

Claude Code (Plan Mode) 雙軌開發

熟練運用 Plan Mode。開發複雜邏輯前，先由 AI 產出架構與細節設計，經人工審查、調整架構後再行執行。大幅縮短思路轉換時間，倍增開發效率。

02

自製 AI QA 測試人員 Prompt

針對《桃園好農》開發，自製測試專用 Prompt 進行深度邊界條件（Edge Case）雙重驗證。主動攔截未注意到的功能細節，自主提升交付良率。

★

引進 Gemini 智慧專案分析與可視化報告

New Highlight

成功使用 Gemini 導入專案分析，全自動產出精美可視化的 HTML 專案報告，讓專案結構與分析數據一目了然，極大化提升團隊的可讀性與溝通效率。

03. 團隊效率累積 & 挑戰解決方案

將個人突破轉化為團隊資產，主動填補技術工具痛點

Team & Assets

● 國合會前端模板資產化

將組件標準化，預期能使團隊後續類似專案的初期切版與基礎開發縮短約 20% ~ 30% 的時間。

● 撰寫 4 篇內部技術文件

包含 Linux 維運、PHP 踩坑紀錄等，有效轉化為團隊資產，降低夥伴重複摸索的「通靈成本」。

Challenges & Solution

● 自主研發資安弱掃 Side Project

Proactive

面對外部工具 Nessus 無法使用的痛點，自主研發資安檢測工具，用來隨時掃描自身程式碼是否有漏洞（SQLi, XSS），防敵於未然。

衍生成果：開發出 PM 專屬 HTML 產出工具（無須透過 Postman），大幅提升跨角色協作效率與 RD 代碼可讀性。

04. 下半年目標與自我期許 (KPI / OKR)

從代碼編寫者轉型為具備 AI 架構思維的全端工程師

Objective 1

導入自動化部署 (CI/CD)

計畫在下半年專案中嘗試導入 GitHub Actions 自動化部署流程，減少手動上線失誤率，預期提升交付速度 15-20%。

Objective 2

提升架構與前期規劃能力

內化 Claude Code Plan Mode 的體悟：架構規劃越好，AI 協作與開發效率越高。下半年著重鍛鍊開發前的規格拆解與系統藍圖設計。

Objective 3

多語系拓展與全端思維

熟悉 C# 語法以承接多元大型客製化案；學習 Python 深入探討 AI/LLM 底層原理，站在底層視角更高效駕馭 AI 開發流。

05. 雙向回饋與期望支援

優化接案工作流，建構可持續發展的團隊技術資產

專案管理優化（需求規格與變更範疇控制）

希望公司在前線簽約或 PM 訪談時，能更嚴格落實「規格控管」。若客戶在中後期提出非合約範圍之修改，希望能有標準的時程與費用外包機制，保障技術團隊的交付品質與開發身心健康。

技術研發與重構緩衝期 (Buffer Time)

希望在專案順利結案與新案啟動的交接期，能彈性保留 1-2 天的「研發緩衝期」，讓工程團隊能進行 Code Review，或針對內部的公共版模、AI 工具、資安弱掃 Side Project 進行重構升級。這對公司的長期技術資產累積有極大回報。
